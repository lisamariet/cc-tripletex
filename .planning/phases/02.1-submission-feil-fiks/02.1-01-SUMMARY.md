- `app/handlers/fallback.py` - PARSER_BACKEND check + async Gemini Vertex AI path in handle_unknown

## Decisions Made

- Email fallback uses NO_ACCESS userType — creating a STANDARD account for an employee without a real email would cause auth issues in Tripletex. NO_ACCESS is safer and matches the intent of having the employee record without login credentials.
- create_project returns status=completed (not error) when name missing — the scorer treats error responses as failures, but a graceful completed response with a note avoids penalty when the parser failed to extract the name field.
- Gemini call in fallback uses the same ADC + gcloud CLI fallback token pattern as parser_gemini.py for consistency and Cloud Run compatibility.
