Legg til `import asyncio` oeverst i tier3.py og implementer asyncio.gather() for uavhengige kall (per D-08, D-09, D-10):

**create_voucher() (ca linje 52):**
- Postings-loopen kaller _lookup_account() sekvensielt for debit og kredit. Samle alle unike account_numbers FOERST, deretter kjoer `asyncio.gather(*[_lookup_account(client, n) for n in unique_accounts])` for aa hente alle IDs parallelt. Map resultatene tilbake.
- Eksempel-moenster:
```python
# Samle unike account numbers
unique_accounts = set()
for p in postings_input:
    unique_accounts.add(int(p.get("debitAccountNumber") or p.get("accountNumber", 0)))
    unique_accounts.add(int(p.get("creditAccountNumber", 0)))
unique_accounts.discard(0)

# Parallell lookup
account_ids = {}
if unique_accounts:
    results = await asyncio.gather(*[_lookup_account(client, n) for n in unique_accounts])
    account_ids = dict(zip(unique_accounts, results))
```

**correct_ledger_error() — korreksjonsvoucher-oppretting:**
- Naar debit og kredit account lookups er uavhengige, parallelliser:
```python
debit_id, credit_id = await asyncio.gather(
    _lookup_account(client, debit_number),
    _lookup_account(client, credit_number),
)
```
- Finn steder i funksjonen der to eller flere _lookup_account()-kall er sekvensielle og uavhengige.

**monthly_closing() — provision-voucher account lookups:**
- Samme moenster som correct_ledger_error — parallelliser uavhengige account-lookups.

**overdue_invoice() — faktura- og leverandoerfaktura-soek:**
- GET /invoice og GET /supplierInvoice (linje 790, 823) er uavhengige:
```python
inv_resp, sinv_resp = await asyncio.gather(
    client.get("/invoice", params={...}),
    client.get("/supplierInvoice", params={...}),
)
```

**VIKTIG per D-10:** IKKE parallelliser POST-kall. Kun GET og _lookup_*-kall som er uavhengige av hverandre.

Feilhaandtering: asyncio.gather() med `return_exceptions=False` (default) — la exceptions propagere normalt.
